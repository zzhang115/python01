	<h2 class="post-title"><div id="title">老王python:django model的get和filter方法的区别</div></h2>
	<div class="post-content">
		<div class="post-body">  django的get和filter方法是django model常用到的，搞清楚两者的区别非常重要。<br/>  <br/>  为了说明它们两者的区别定义2个models<br/>  <br/>  class Student(models.Model):<br/>      name = models.CharField('姓名', max_length=20, default='')<br/>      age = models.CharField('年龄', max_length=20, default='')<br/>  <br/>  <br/>  class Book(models.Model):<br/>  	  student = models.ForeignKey(Student)<br/>  	  <br/>  	  <br/><b>  一.先说下django的get方法：</b><br/>  <br/>  1django的get方法是从数据库的取得一个匹配的结果，返回一个对象，如果记录不存在的话，它会报错。<br/>  比如我数据库里有一条记录，记录的name的值是<a href="http://www.cnpythoner.com" target="_blank">老王python</a>的话，我用student = Student.objects.get(name='老王python')，<br/>  返回的是一个记录对象，你可以通过student.__dict__来查看，它返回的是一个字典的形式，｛'key':valeus｝,key是字段的名称，而values是值的内容。<br/>  而如果我用get方法来查询一个数据库里不存在的记录，程序会报错。<br/>  比如:student = Student.objects.get(name='老王'),你自己可以运行看下。<br/>  <br/>  2如果你用django的get去取得关联表的数据的话，而关键表的数据如果多于2条的话也会报错。<br/>  比如我的student表里有一个记录：<br/>  id   name     age<br/>   1   python  24<br/>   <br/>   book表：<br/>   <br/>   id  student_id<br/>    1      1<br/>    2      1 <br/>    <br/>   我用<br/>   student = Student.objects.get(name='python')<br/>   book = Book.objects.get(student)<br/>   它也会报错，因为book表有2条记录和student表相匹配。<br/>   <br/>   <br/>   <b>二.再说下django filter:</b><br/>   <br/>  1django的filter方法是从数据库的取得匹配的结果，返回一个对象列表，如果记录不存在的话，它会返回[]。<br/>   比如我数据库里有一条记录，记录的name的值是老王python的话，我用student = Student.objects.filter(name='老王python')<br/>   它返回的student是一个对象的列表，可以看的出来student[0]和上面的get方式返回的student的结果是一样的。<br/>   <br/>   2如果你用django的get去取得关联表的数据的话，无论关联表有多少记录的都不会报错。<br/>   django 除了model比较强大的话，表单和模板也很强大，想要了解表单的可以看看：<br/> <a target="_blank"  href="http://www.cnpythoner.com/post/13.html">django forms DateTimeField</a> <br/><a target="_blank"  href="http://www.cnpythoner.com/post/7.html">django form 显示</a><br/><a target="_blank"  href="http://www.cnpythoner.com/post/6.html">django form 验证</a><br/> <a target="_blank"  href="http://www.cnpythoner.com/post/15.html">django template</a><br/>   <br/>   另外我从别的资料里看到filter好像有缓存数据的功能，第一次查询数据库并生成缓存，下次再调用filter方法的话，直接取得缓存的数据，会get方法每次执行都是直接查询数据库的，不知道这个是不是正确。<br/>  <br/>  这是我用了一段时间django的一点心得体会，希望对大家了解django get和filter有帮助！<br/> <link rel="stylesheet" href="http://www.cnpythoner.com/PLUGIN/Rating/rating.css" type="text/css" /><div id="RatingPre"></div><Script
