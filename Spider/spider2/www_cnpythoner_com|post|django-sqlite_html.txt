	<h2 class="post-title"><div id="title">django sqlite数据库配置问题</div></h2>
	<div class="post-content">
		<div class="post-body">  前几天有一个朋友发了一个django方面的问题给我。具体是：<br/><br/>  请问我在你给的那本《Django实战练习》中第二章 configuring your first Django project ，一步步按他的说明做完，由于我在、安装Django的文件夹有汉字出现错误，所以我就在同一个盘下见了个文件夹 Django-projects，setting中的设置是'NAME': 'D:\Django-projects'。<br/>最后运行时出现<br/><br/>Unable to open database file<br/>的错误。看完网上的说法修改了文件夹的写入读出权限，可还是同样错误。<br/>不知为什么。<br/>我用的是Django的最后一个版本2.7.1，不知道是不是根本就没有自带SQLite？<br/><br/>解决思路：<br/>报错：Unable to open database file 说明程序打不开文件，可以有两种情况：<br/>1程序没有权限打开数据库所在的文件夹，如果你是在windows下做开发的话，那就不存在这个问题了。<br/>2数据库路径不对。<br/><br/>这个朋友是在windows下做开发的，所以不存在第一种情况了，报错应该是数据库路径设置不正确。<br/><br/>解决办法：<br/><br/>import os<br/>DEBUG = True<br/>TEMPLATE_DEBUG = DEBUG<br/><br/>SITE_PATH = os.path.dirname(os.path.abspath(__file__))#系统路径 我们可以自己定义好路径<br/><br/>ADMINS = (<br/>    # ('Your Name', 'your_email@domain.com'),<br/>)<br/><br/>MANAGERS = ADMINS<br/><br/>DATABASE_ENGINE = 'sqlite3'           # 'postgresql_psycopg2', 'postgresql', 'mysql', 'sqlite3' or 'oracle'.<br/>DATABASE_NAME = os.path.join(SITE_PATH, 'guestbook.db')  数据库文件放在文件夹的根目录位置。<br/><br/>顺便说下python 默认是带sqlite数据库的。<br/><br/>希望通过这个例子，对大家避免一些错误有帮助。<br/><br/><link rel="stylesheet" href="http://www.cnpythoner.com/PLUGIN/Rating/rating.css" type="text/css" /><div id="RatingPre"></div><Script language="javascript" type="text/javascript">var PostID=157;</script><Script language="javascript" type="text/javascript" src="http://www.cnpythoner.com/PLUGIN/Rating/Rating.js"></script> </div>
